# TON Agentic Wallet

## Overview
В данном репозитории реализованы смарт-контракты для TON Agentic Wallets — кошельков, созданных для управления AI-агентами. Агенты могут совершать любые транзакции в блокчейне TON (используя TON MCP Server), их кошельки обладают всем функционалом W5. Кошельки агентов привязаны к адресу пользователя, и он может в любой момент перехватить контроль над ними. Для удобства индексации каждый кошелек является SBT из определенной коллекции.

Референсы: [Wallet V5](https://github.com/ton-blockchain/tolk-bench/tree/master/contracts_Tolk/05_wallet-v5), [NFT](https://github.com/ton-blockchain/tolk-bench/tree/master/contracts_Tolk/02_nft)

## Деплой
Деплой происходит при первом депозите на кошелек, не заставляя юзера делать дополнительные действия. Есть 2 возможных флоу создания Agentic Wallet:
1. Кошелек со сгенерированным агентом operatorPublicKey создается пользователем. На кошельке проверяется, что адрес отправителя совпадает с сохраненным адресом владельца
2. Агент, имея как минимум один кошелек, созданный пользователем, может создать дополнительные кошельки. Для этого он отправляет транзакцию со своего кошелька на новый. В транзакции передаются данные для проверки корректности адреса отправителя (masterAddress, operatorPublicKey). Новый кошелек рассчитывает stateInit и адрес отправителя и сверяет его с фактическим.
В случае непрохождения проверок адреса, кошелек деплоется в состоянии uninitialized, и не отображается в эксплорерах. Это делает невозможным создание "нежелательных" кошельков для юзера (спама).
Индекс NFT рассчитывается как хэш от (masterAddress, operatorPublicKey). 

## Функции контракта
- Агент может использовать весь функционал Wallet V5, подписывая транзакции с помощью operatorKey
- Пользователь может управлять кошельком через internal сообщения, как неудаляемый extension 
- Пользователь может передать управление кошельком другому агенту, изменив operatorPublicKey, или деактивировать агента (установив operatorPublicKey = 0)
  
## Лимиты агента
На данный момент не используется никаких лимитов на транзакции для агентов ("Fund what you risk"). Это осознонное решение для упрощения первой версии контрактов.
